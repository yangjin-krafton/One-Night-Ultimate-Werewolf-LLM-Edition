# 시나리오 생성 규격

이 문서는 앞으로 이 프로젝트에서 `scenarios/*.json` 과 `scenarios_tts/*.tts.json` 을 새로 만들거나 수정할 때 따라야 하는 공통 규격이다.

기준 문서:

- 규칙 기준: [나무위키.md](/mnt/d/Weeks/One-Night-Ultimate-Werewolf-LLM-Edition/docs/나무위키.md)
- 기존 생성 가이드: [LLM 스토리 생성 지침.md](/mnt/d/Weeks/One-Night-Ultimate-Werewolf-LLM-Edition/docs/LLM%20%EC%8A%A4%ED%86%A0%EB%A6%AC%20%EC%83%9D%EC%84%B1%20%EC%A7%80%EC%B9%A8.md)

이 문서의 목적:

- 더 이상 `에피소드 2개 고정`으로 시나리오를 만들지 않는다.
- 모든 신규 시나리오는 `ep1`, `ep2`, `ep3`를 기본 제공한다.
- `ep1`, `ep2`, `ep3`는 스토리 분량 차이보다 `밤 진행 안내 밀도`의 차이를 뜻한다.
- 시나리오의 테마도 `초보자형`과 `숙련자형`으로 나눠 설계한다.
- 역할 풀은 최신 [나무위키.md](/mnt/d/Weeks/One-Night-Ultimate-Werewolf-LLM-Edition/docs/%EB%82%98%EB%AC%B4%EC%9C%84%ED%82%A4.md) 기준 전체 범위를 지원 대상으로 본다.

## 1. 기본 원칙

### 1.1 에피소드 의미 재정의

이제 이 프로젝트에서 `episode`는 반드시 `스토리 챕터`를 뜻하지 않는다.

신규 시나리오 기본 규칙:

- `ep1`: 초보자용 상세 밤 진행
- `ep2`: 중간 길이 표준 진행
- `ep3`: 숙련자용 빠른 진행

즉, 같은 역할 덱과 같은 규칙이라도 `나레이션 밀도`를 세 단계로 만들어야 한다.

### 1.2 모든 시나리오의 최소 구성

신규 시나리오는 최소한 아래를 가져야 한다.

1. `ep1`
2. `ep2`
3. `ep3`

권장 해석:

- `ep1`, `ep2`, `ep3`는 같은 시나리오의 `튜토리얼 밀도별 버전`
- 스토리 전개가 필요한 경우에도, 최소한 밤 안내 톤은 이 세 단계 기준을 따라야 한다

### 1.3 역할 범위

역할 풀은 기본판만 기준으로 삼지 않는다.

지원 대상 역할 범위:

- 기본판
- Daybreak
- Vampire
- Alien
- 각 보너스 팩에서 언급된 추가 역할

즉, 앞으로 시나리오 생성 규칙은 `최신 나무위키 정리본에 나온 역할 전체`를 포함하는 방향으로 작성한다.

### 1.4 전체 역할 커버리지와 랜덤 덱 구성

절대 원칙:

- TTS 시나리오의 `roleClips`는 시나리오가 지원하는 **모든 역할**에 대해 나레이션 클립을 포함해야 한다
- 런타임 시나리오는 인원수별 고정 덱(`roleDeck`)을 정의하지 않는다
- 대신 전체 역할 풀(`rolePool`)과 밤 기상 순서(`roleWakeOrder`)를 정의한다
- 게임 엔진이 참가 인원수에 따라 역할 풀에서 랜덤으로 덱을 구성한다

이유:

- 같은 시나리오라도 매 판마다 다른 역할 조합이 나와야 리플레이 가치가 높아진다
- TTS 클립이 일부 역할만 포함하면, 랜덤 구성 시 나레이션이 빠지는 역할이 생긴다
- 역할 풀 전체에 클립을 만들어 두면, 어떤 조합이 나와도 밤 진행이 가능하다

덱 구성 기본 규칙:

- 플레이어 수 N명일 때, 덱은 N+3장이다 (중앙 카드 3장 포함)
- 게임 엔진이 `rolePool`에서 N+3장을 뽑아 덱을 만든다
- 선택된 역할 중 밤 행동이 있는 역할만 `roleWakeOrder` 순서에 따라 밤에 호출된다
- 밤 행동이 없는 역할(마을주민, 사냥꾼, 무두장이 등)은 호출하지 않는다

## 2. 시나리오 타입

모든 시나리오는 먼저 아래 둘 중 하나로 분류한다.

### 2.1 초보자형 시나리오

목적:

- 밤 행동 규칙을 배우게 하는 것
- 역할별 행동 차이를 몸으로 익히게 하는 것
- 카드 이동, 공개, 표식, 팀 변화 개념을 단계적으로 이해시키는 것

특징:

- 규칙 설명과 안내가 많다
- 밤 행동 직전 주의사항을 다시 말해 준다
- 정보 역할과 교환 역할을 명확히 구분해서 넣는다
- 혼란 유발 역할은 소수만 넣는다
- 독립 승리, 표식, 유물, 앱 연동 역할은 뒤로 미룬다

권장 테마:

- 수련회
- 마을 집회
- 신입 조사팀
- 야간 실습
- 방송 리허설

금지 경향:

- 분위기만 강조하고 규칙 설명이 빈약한 구성
- 첫 판부터 표식/유물/제3세력/앱 의존 역할을 대량 혼합

### 2.2 숙련자형 시나리오

목적:

- 규칙 설명보다 역할 연기와 추론전을 살리는 것
- 짧은 밤 안내 뒤 바로 토론과 속임수로 넘어가게 하는 것
- 같은 규칙이라도 분위기와 캐릭터성으로 몰입을 높이는 것

특징:

- 설명보다 분위기와 테마 연출 비중이 높다
- 밤 멘트는 짧고 정확해야 한다
- 플레이어가 이미 기본 규칙을 안다고 가정한다
- 표식, 진영 변화, 특수 승리 조건이 들어가도 된다

권장 테마:

- 우주 정거장
- 폐교 촬영팀
- 야간 병동 인수인계
- 흡혈귀 살롱
- 외계 신호 관측소

금지 경향:

- 역할 행동을 장황하게 다시 가르치는 것
- 긴장감 대신 설명문처럼 읽히는 대사

## 3. 에피소드 단계 규격

### 3.1 `ep1` 초보자용 상세 진행

목적:

- 처음 하는 사람도 밤 순서를 따라갈 수 있게 한다
- 각 역할의 행동 방법과 실수 방지 포인트를 충분히 준다

대사 원칙:

- 가장 길다
- 역할 호출 전후 맥락을 설명한다
- 행동 방법과 주의사항을 함께 말한다
- 실수 방지 문장이 반드시 들어간다

권장 분량:

- `openingClips`: 4~7문장
- 역할 `during`: 4~6문장
- 역할 `after`: 1~2문장
- `nightOutroClips`: 2~4문장

설명 밀도:

- 누가 눈을 뜨는지 명확히 말한다
- 무엇을 선택하는지 말한다
- 본인 카드/남 카드/중앙 카드 중 무엇을 만지는지 말한다
- 확인 후 다시 눈 감는 타이밍을 말한다
- 예외가 있으면 짧게 알려 준다

예시 방향:

- `강도, 눈을 뜨세요. 다른 플레이어 한 명의 카드와 자신의 카드를 바꾸세요. 바꾼 뒤에는 새로 가져온 자기 카드를 혼자 확인합니다. 다른 사람에게는 보여주지 마세요. 확인이 끝났으면 다시 눈을 감으세요.`

### 3.2 `ep2` 중간 길이 표준 진행

목적:

- 규칙을 조금 아는 사람에게는 지루하지 않게
- 그래도 헷갈리는 행동은 놓치지 않게

대사 원칙:

- `ep1`보다 짧다
- 행동 핵심만 남긴다
- 불필요한 분위기 묘사는 줄인다

권장 분량:

- `openingClips`: 2~4문장
- 역할 `during`: 2~4문장
- 역할 `after`: 1문장
- `nightOutroClips`: 1~2문장

설명 밀도:

- 행동 대상
- 확인 여부
- 교환 여부
- 눈 감는 마감

예시 방향:

- `강도, 눈을 뜨세요. 다른 플레이어 한 명과 카드를 바꾸고, 바꾼 자신의 새 카드를 확인하세요. 끝났으면 눈을 감으세요.`

### 3.3 `ep3` 숙련자용 빠른 진행

목적:

- 익숙한 플레이어가 리듬을 끊지 않고 진행하게 한다
- 밤 진행 시간을 최소화한다

대사 원칙:

- 가장 짧다
- 설명보다 명령형이 우선이다
- 이미 알고 있다는 전제로 진행한다

권장 분량:

- `openingClips`: 1~2문장
- 역할 `during`: 1~2문장
- 역할 `after`: 생략 가능, 있으면 1문장
- `nightOutroClips`: 1문장

설명 밀도:

- 호출
- 행동
- 종료

예시 방향:

- `강도, 눈 뜨세요. 한 명과 교환하고 새 카드를 확인하세요.`

## 4. TTS 분량 규격

모든 시나리오는 같은 역할이라도 `ep1`, `ep2`, `ep3`에서 대사량이 명확히 달라야 한다.

### 4.1 절대 원칙

- 같은 역할 대사를 세 에피소드에 복붙하지 않는다
- `ep1 > ep2 > ep3` 순서로 정보량이 줄어들어야 한다
- 줄어드는 것은 `설명량`이지 `핵심 규칙 정확성`이 아니다

### 4.2 역할별 분량 가이드

#### 정보 확인형 역할

예:

- seer
- apprentice_seer
- paranormal_investigator
- marksman
- psychic

규칙:

- `ep1`: 무엇을 몇 장 볼 수 있는지 풀어 설명
- `ep2`: 볼 수 있는 대상과 장수만 간단히 설명
- `ep3`: 행동 명령만 남김

#### 교환형 역할

예:

- robber
- troublemaker
- drunk
- witch
- village_idiot
- gremlin
- rascal

규칙:

- `ep1`: 누구 카드가 바뀌는지, 본인이 결과를 아는지까지 설명
- `ep2`: 교환 대상과 확인 여부만 설명
- `ep3`: 교환 명령만 남김

#### 팀 확인형 역할

예:

- werewolf
- minion
- mason
- vampire
- alien
- squire
- beholder

규칙:

- `ep1`: 누구를 확인하는지, 누가 자신을 모르는지도 설명 가능
- `ep2`: 확인 대상만 간단히 전달
- `ep3`: `눈 떠라 / 확인해라 / 눈 감아라` 수준으로 압축

#### 특수 승리/표식형 역할

예:

- tanner
- cursed
- cupid
- instigator
- assassin
- apprentice_assassin
- diseased

규칙:

- `ep1`에서는 승리 조건에 영향을 주는 점을 짧게 언급 가능
- `ep2`에서는 행동만 설명하고 승리 조건은 거의 생략
- `ep3`에서는 밤 행동이 필요한 내용만 남김

## 5. 시나리오 테마 규격

### 5.1 초보자형 테마 규칙

초보자형 시나리오는 `세계관`보다 `규칙 이해`가 우선이다.

반드시 지켜야 할 것:

- 밤 멘트에서 행동 대상을 명확히 말할 것
- 역할 간 차이를 쉽게 느끼게 할 것
- 테마 문장이 규칙 문장을 가리지 않게 할 것
- 오프닝도 짧고 분명해야 할 것

권장 역할 축:

- 기본판 중심
- Daybreak 일부
- 공개/표식/유물/앱 의존 역할 최소화

좋은 예:

- `신입 야간 경비 교육`
- `마을 첫 야간 순찰`
- `초심자용 추리 캠프`

나쁜 예:

- 규칙보다 설정 고유명사가 더 많은 시나리오
- 역할 행동보다 배경 서사가 더 길게 읽히는 시나리오

### 5.2 숙련자형 테마 규칙

숙련자형 시나리오는 `테마 연기`와 `분위기` 비중을 높여도 된다.

반드시 지켜야 할 것:

- 규칙 문장은 짧아도 정확해야 한다
- 각 역할의 행동을 흐리지 않는 범위에서 테마를 넣을 것
- 긴 설명 대신 장면 전환과 캐릭터 톤으로 분위기를 만들 것

권장 역할 축:

- 늑대 진영 변형
- 뱀파이어 확장
- 에일리언 확장
- 독립 승리 역할
- 표식 기반 역할

좋은 예:

- `폐쇄 병동의 야간 인수인계`
- `흡혈귀 귀족의 가면 만찬`
- `오염된 관측소의 외계 신호`

나쁜 예:

- 빠른 진행이어야 하는데 역할 설명이 튀어나오는 시나리오
- 숙련자용인데 초보자 튜토리얼을 반복하는 시나리오

## 7. 역할 풀 구성 규칙

역할 풀(`rolePool`)은 시나리오가 지원하는 **전체 역할 목록**이다.
게임 엔진이 인원수에 따라 이 풀에서 랜덤으로 덱을 구성하므로, 풀 자체의 균형이 중요하다.

### 7.1 초보자형 시나리오 풀

권장 구성:

- 기본판 역할 위주로 풀을 채운다
- 정보 역할, 교환 역할, 팀 확인 역할을 고루 포함한다
- 독립 승리 역할은 최대 1~2종까지만 풀에 넣는다
- 풀 전체 규모는 10~15장 범위가 적당하다

피해야 할 풀 구성:

- `village_idiot` + `gremlin` + `witch` 같이 이동량이 큰 역할 다중 포함
- `vampire`, `alien`, `tanner`, `assassin`이 모두 한 풀에 들어가는 구성
- 앱 의존 역할을 튜토리얼 시나리오 풀에 대량 투입

### 7.2 숙련자형 시나리오 풀

가능한 방향:

- 진영 변형 역할을 풀에 섞는다
- 팀 판정이 흔들리는 역할을 넣는다
- 토론 심리전을 강하게 만드는 역할을 선택한다
- 표식과 특수 승리 조건 역할을 적극 포함한다
- 풀 규모를 크게 잡아 매 판 조합 변동을 극대화할 수 있다

주의:

- 복잡성을 올려도 밤 순서는 반드시 명확히 선언해야 한다
- TTS 대사는 짧아져도 역할 처리 규칙이 모호하면 안 된다
- **풀에 포함된 모든 밤 행동 역할에 대해 TTS 클립이 존재해야 한다**

## 8. JSON 작성 규칙

### 8.1 런타임 시나리오

`scenarios/*.json`은 다음 성격을 가진다.

- 어떤 역할이 풀에 들어가는지 (`rolePool`)
- 밤에 어떤 순서로 호출하는지 (`roleWakeOrder`)
- 초보자형/숙련자형 시나리오인지 (`scenarioMode`)
- 에피소드별 나레이션 밀도 정책

핵심 원칙:

- 런타임 시나리오는 **인원수별 고정 덱을 정의하지 않는다**
- 대신 `rolePool`에 사용 가능한 역할 전체와 수량을 정의한다
- 게임 엔진이 인원수에 따라 `rolePool`에서 랜덤으로 N+3장을 뽑아 덱을 구성한다
- `roleWakeOrder`는 풀에 포함된 모든 밤 행동 역할의 호출 순서를 정의한다

예시:

```json
{
  "schemaVersion": 2,
  "scenarioId": "dark_citadel",
  "scenarioMode": "advanced",
  "episodeProfiles": {
    "ep1": "tutorial_long",
    "ep2": "standard_mid",
    "ep3": "expert_fast"
  },
  "rolePool": [
    "werewolf", "werewolf", "mystic_wolf", "minion",
    "tanner", "villager", "villager", "villager",
    "seer", "paranormal_investigator", "robber", "witch",
    "troublemaker", "drunk", "insomniac", "curator"
  ],
  "roleWakeOrder": [
    "werewolf", "mystic_wolf", "minion",
    "seer", "paranormal_investigator",
    "robber", "witch", "troublemaker",
    "drunk", "curator", "insomniac"
  ],
  "narration": {
    "playbackPolicy": {
      "narratorAllowedPhases": ["episode_start"],
      "silentPhases": ["debate", "vote", "vote_result", "reveal"]
    },
    "openingClipCount": 1,
    "nightOutroClipCount": 1,
    "roleClipPolicy": {
      "default": { "before": 0, "during": 1, "after": 1 },
      "ep3Override": { "before": 0, "during": 1, "after": 0 }
    },
    "roleClipOverrides": {
      "minion": { "before": 0, "during": 1, "after": 0 }
    }
  }
}
```

`scenarioMode` 허용값:

- `beginner`
- `advanced`

`rolePool` 작성 규칙:

- 같은 역할이 여러 장 필요하면 배열에 중복 포함한다 (예: `"werewolf"` 2회)
- 밤 행동이 없는 역할(마을주민, 사냥꾼, 무두장이 등)도 풀에 포함한다
- `roleWakeOrder`에는 밤 행동이 있는 역할만 순서대로 나열한다

### 8.2 TTS 시나리오

`scenarios_tts/*.tts.json`은 반드시 `ep1`, `ep2`, `ep3`를 기본 포함한다.

핵심 원칙:

- `roleClips`는 `rolePool`에 포함된 **밤 행동이 있는 모든 역할**에 대해 나레이션 클립을 포함해야 한다
- 게임 엔진이 랜덤으로 덱을 구성하므로, 어떤 역할이 선택되더라도 나레이션이 존재해야 한다
- 일부 역할만 클립을 만들면 안 된다

예시:

```json
{
  "schemaVersion": 2,
  "scenarioId": "dark_citadel",
  "scenarioMode": "advanced",
  "episodes": {
    "ep1": {
      "openingClips": "성채의 철문이 닫혔다. ...",
      "roleClips": {
        "werewolf": {
          "during": "늑대인간, 눈을 떠라. ...",
          "after": "늑대인간은 눈을 감아라. ..."
        },
        "mystic_wolf": { "during": "...", "after": "..." },
        "minion": "하수인, 눈을 떠라. ...",
        "seer": { "during": "...", "after": "..." },
        "paranormal_investigator": { "during": "...", "after": "..." },
        "robber": { "during": "...", "after": "..." },
        "witch": { "during": "...", "after": "..." },
        "troublemaker": { "during": "...", "after": "..." },
        "drunk": { "during": "...", "after": "..." },
        "curator": { "during": "...", "after": "..." },
        "insomniac": { "during": "...", "after": "..." }
      },
      "nightOutroClips": "밤이 물러간다. ..."
    },
    "ep2": { "..." : "ep1보다 짧게, 동일한 역할 전체 포함" },
    "ep3": { "..." : "명령형 압축, 동일한 역할 전체 포함" }
  }
}
```

## 9. 밤 진행 문장 작성 규칙

### 9.1 `ep1`

- `왜 이 행동을 하는지`까지 짧게 알려줘도 된다
- `실수하지 마라` 계열 문장을 넣어도 된다
- `자기 카드인지 남의 카드인지 중앙 카드인지`를 꼭 구분한다

### 9.2 `ep2`

- 설명은 줄이되, `선택 대상`과 `확인 여부`는 남긴다
- 한 역할당 2~4문장을 넘기지 않는다

### 9.3 `ep3`

- 연기보다 리듬이 우선
- 군더더기 없는 명령형
- 동일 역할이라도 `ep1` 표현을 줄인 요약문이어야 한다

## 10. 나레이션 스타일 규칙

### 10.1 초보자형

- 말투는 분명하고 친절하게
- 지나친 은유, 시적 표현 금지
- 설명문이 곧 규칙 전달이 되게 작성

### 10.2 숙련자형

- 설정 몰입, 톤, 긴장감 강화 가능
- 다만 행동 규칙 문장은 여전히 명확해야 한다
- 분위기 문장과 행동 문장을 섞더라도, 행동 문장이 항상 우선이다

## 11. 생성 시 체크리스트

새 시나리오를 만들 때는 아래를 확인한다.

1. `ep1`, `ep2`, `ep3`가 모두 있는가
2. `ep1 > ep2 > ep3` 순으로 설명량이 실제로 줄어드는가
3. 시나리오가 `beginner`인지 `advanced`인지 명시했는가
4. 역할 ID가 표준 ID를 쓰는가
5. 밤 행동 순서가 역할 구성과 충돌하지 않는가
6. 초보자형인데 지나치게 복잡한 확장 역할을 많이 넣지 않았는가
7. 숙련자형인데 설명이 너무 길어 리듬을 해치지 않는가
8. TTS 대사가 규칙보다 분위기에 치우쳐 있지 않은가
9. **`rolePool`에 포함된 밤 행동 역할 전체에 대해 TTS 클립이 존재하는가**
10. **런타임 시나리오가 고정 덱이 아닌 `rolePool` 방식으로 작성되었는가**

## 12. 권장 운영

실제 작업 순서는 아래가 안전하다.

1. 먼저 `scenarioMode`를 정한다
2. `rolePool` 전체를 확정한다 (수량 포함)
3. `roleWakeOrder`를 풀의 모든 밤 행동 역할 기준으로 확정한다
4. `ep1`을 가장 자세하게 쓴다 — **풀의 모든 밤 행동 역할에 대해 클립을 작성한다**
5. `ep1`을 줄여 `ep2`를 만든다 — 동일한 역할 전체를 포함하되 분량을 줄인다
6. `ep2`를 더 줄여 `ep3`를 만든다 — 동일한 역할 전체를 포함하되 명령형으로 압축한다
7. 런타임 시나리오에 `rolePool`과 `roleWakeOrder`를 작성한다
8. 마지막에 음성 매핑과 TTS 생성 가능 여부를 확인한다

이 순서를 지키면 `빠른 버전부터 먼저 써서 규칙이 빠지는 문제`와 `일부 역할 클립 누락 문제`를 모두 줄일 수 있다.

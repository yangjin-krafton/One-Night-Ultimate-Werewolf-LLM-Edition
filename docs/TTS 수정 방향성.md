# TTS 수정 방향성

## 목적

이 문서는 현재 `public/assets/scenarios_tts/*.tts.json`의 TTS 대사를 다음 방향으로 정리하기 위한 기준이다.

- `늑대를 찾아라`라는 게임의 핵심 목표를 흐리지 않는다.
- 테마는 `줄거리 이해`가 아니라 `말투, 태도, 감각, 토론 분위기`로 살린다.
- 밤 대사는 플레이어가 다음 날 바로 써먹을 수 있는 `심리 프레임`과 `사회적 압력`을 남긴다.
- `ep1`, `ep2`, `ep3`는 스토리 챕터보다 `진행 밀도 차이`가 먼저 느껴지게 한다.

## 현재 문제 요약

- 숙련자형 시나리오가 사건 설명에 너무 많은 비중을 둔다.
- 역할별 대사가 `행동 지시`보다 `배경 독백`으로 들리는 구간이 많다.
- 플레이어는 테마를 이해해야 할 것 같은 부담을 느끼지만, 그 이해가 승리에 직접 연결되지는 않는다.
- 결과적으로 `대사는 다양하지만 게임 재미를 직접 올리지는 못한다`는 인상이 생긴다.

## 수정 원칙

1. `플롯`보다 `프레임`을 남긴다.
2. 각 역할은 `내가 누구처럼 말하는가`가 먼저 보여야 한다.
3. 각 역할 대사는 `행동 지시 + 태도 + 낮 토론용 훅` 구조를 기본으로 한다.
4. 사건 진실을 길게 깔지 말고, 플레이어가 다음 날 사용할 수 있는 압박 문장으로 바꾼다.
5. 같은 역할이라도 시나리오마다 `감각 어휘`만 바꾸고, 역할의 사회적 기능은 유지한다.

## 금지 경향

- 사건의 전말을 밤에 이미 설명해 버리는 대사
- 역할 고유 행동보다 분위기 묘사가 더 긴 대사
- 모든 역할이 비슷한 독백 톤으로 말하는 구성
- `ep1`, `ep2`, `ep3`의 길이와 밀도가 거의 같은 구성
- 무행동 역할이 아무 사회적 힌트도 못 남기는 구성

## 권장 템플릿

- `openingClips`: 상황 소개 1개 + 오늘 토론의 의심 축 1개
- `roleClips`: 역할 개성 도입 1개 + 개성 있는 호출 1개 + 행동/규칙 1개 + 개성 있는 종료 1개
- `nightOutroClips`: 토론 시작 문장 대신, 의심 방향을 묻는 질문형 문장

예시:

- 나쁨: `그 밤의 복도에는 젖은 발자국과 오래된 죄책감이 남아 있었다.`
- 좋음: `강도, 한 명과 바꾸고 새 카드를 확인해라. 오늘은 누가 먼저 입을 여는지보다, 누가 너무 빨리 정답을 말하는지를 기억해라.`

## 반복 대사 포맷 원칙

반복 대사는 단순한 안내 멘트가 아니라, 각 역할의 말투 차별성이 가장 안정적으로 드러나는 구간이다. 따라서 역할 TTS는 아래 순서를 기본 포맷으로 고정하고, 각 단계의 어휘와 태도를 역할별로 다르게 써야 한다.

기본 흐름:

1. `개성 대사`로 역할의 성격을 먼저 찍는다.
2. `개성 있는 눈을 떠라`로 호출한다.
3. `행동`을 짧고 분명한 동사로 지시한다.
4. `플레이 규칙 상세`를 필요한 범위에서만 덧붙인다.
5. `개성 있는 눈을 감아라`로 종료를 호출한다.
6. `개성 있는 마무리 대사`로 낮 토론용 인상이나 압박을 남긴다.

포맷 해석:

- `눈을 떠라`, `눈을 감아라` 같은 반복 문구도 역할마다 같은 톤으로 복붙하지 않는다.
- 행동 지시 전후의 짧은 개성 문장이 역할 정체성을 만든다.
- 규칙 설명은 행동을 보조하는 선에서만 넣고, 분위기 연기는 시작과 마무리에 몰아준다.
- 무행동 역할도 `개성 대사 > 개성 있는 눈을 떠라 > 규칙/사회적 임무 > 개성 있는 눈을 감아라 > 마무리` 흐름은 유지한다.
- `beginner_dark_fantasy`만 예외적으로 개성보다 명확한 규칙 전달을 우선하지만, 그래도 호출과 종료의 문장 리듬은 역할별로 구분되는 편이 좋다.

짧은 예시 포맷:

- `개성 대사`: `강도는 남의 주머니보다 표정을 먼저 뒤진다.`
- `개성 있는 눈을 떠라`: `강도여, 조용히 눈을 떠라.`
- `행동`: `아무 플레이어 한 명과 카드를 바꿔라.`
- `플레이 규칙 상세`: `바꾼 뒤에는 반드시 자기 카드를 확인해라.`
- `개성 있는 눈을 감아라`: `강도여, 들키기 전에 눈을 감아라.`
- `개성 있는 마무리 대사`: `내일은 누가 너무 쉽게 남의 정체를 말하는지 봐라.`

## 역할별 밤 행동 글자수 예산 지침

`docs/시나리오 생성 규격.md`의 `4.3 역할별 밤 행동 권장 글자수 예산`은 이 문서에서도 그대로 따른다. 다만 실제 TTS 수정 작업에서는 아래처럼 해석하는 편이 더 실용적이다.

핵심 해석:

- 글자수 예산은 `최대치 제한`이 아니라 `최소 행동 시간 보장용 하한선`이다.
- 예산을 넘겨도 괜찮다. 행동이 명확하고 실제 조작 시간을 충분히 보장한다면 초과는 문제로 보지 않는다.
- 너무 짧은 대사가 더 위험하다. 플레이어가 카드를 뒤집거나 교환하기 전에 다음 호출이 들어올 수 있기 때문이다.
- 연기 톤을 넣더라도 행동 동사가 흐려지면 안 된다.
- 복잡한 역할일수록 긴 문장보다 `짧은 문장 + 명확한 동사`가 우선이다.
- `ep3`도 빠른 진행일 뿐, 물리 조작 시간이 사라지는 것은 아니다.

실무 규칙:

1. 먼저 역할의 실제 손동작 시간을 기준으로 문장 길이를 잡는다.
2. 그다음 테마 말투를 덧입힌다.
3. 분위기 문장은 행동 지시 뒤에 붙이고, 1문장 이내로 제한한다.
4. 예산 초과 자체는 허용한다. 다만 길어질수록 행동 동사와 규칙 전달이 흐려지지 않는지 확인한다.
5. 예산을 초과한 대사보다 `예산에 한참 못 미치는 대사`를 먼저 경계한다.

복잡도별 하한 가이드:

| 복잡도 | ep1 | ep2 | ep3 | 대표 행동 |
| --- | --- | --- | --- | --- |
| 단순 | 48~60자 | 30~42자 | 18~30자 | 확인만, 물리 조작 없음 |
| 기본 | 72~90자 | 48~60자 | 30~42자 | 카드 1장 확인 |
| 표준 | 90~108자 | 60~72자 | 42~48자 | 카드 교환 1회 또는 선택적 확인 |
| 복잡 | 120~150자 | 72~90자 | 48~60자 | 다단계 조작, 전체 이동 |
| 가변 | 150자+ | 108자+ | 72자+ | 복제 후 추가 행동 |

자주 쓰는 역할 예산 요약:

| 역할군 | 대표 역할 | ep1 | ep2 | ep3 | 메모 |
| --- | --- | --- | --- | --- | --- |
| 팀 확인형 | `werewolf`, `minion`, `mason`, `beholder` | 60~90자 | 42~60자 | 30~42자 | 확인 대상이 명확해야 한다 |
| 정보 확인형 | `seer`, `apprentice_seer`, `aura_seer` | 72~90자 | 48~60자 | 30~42자 | 몇 장, 누구를 보는지 분명해야 한다 |
| 복합 정보형 | `paranormal_investigator` | 120자 | 90자 | 60자 | 멈춤 조건, 전향 조건이 있으므로 여유가 필요하다 |
| 교환형 | `robber`, `troublemaker`, `drunk` | 72~90자 | 48~60자 | 30~42자 | 교환 후 확인 여부를 빠뜨리면 안 된다 |
| 선택 교환형 | `witch` | 120자 | 72자 | 48자 | 확인 후 선택 교환까지 포함해야 한다 |
| 전체 이동형 | `village_idiot` | 120자 | 90자 | 60자 | 실제 조작 시간이 가장 긴 축에 속한다 |
| 공개형 | `revealer` | 72자 | 48자 | 30자 | 공개 후 되돌리기 조건이 드러나야 한다 |
| 마지막 확인형 | `insomniac` | 60자 | 42자 | 30자 | 밤의 마지막 확인이라는 점이 중요하다 |
| 가변형 | `doppelganger` | 150자+ | 108자+ | 72자+ | 복제 후 추가 행동 때문에 가장 여유가 필요하다 |

시나리오 수정 시 체크 포인트:

- `beginner_dark_fantasy`는 규칙형 시나리오이므로 예산 하한을 넉넉하게 지키는 편이 안전하다.
- 분위기형 시나리오도 `alpha_wolf`, `witch`, `village_idiot`, `doppelganger`, `paranormal_investigator`는 절대 과도하게 줄이지 않는다.
- 반대로 `dream_wolf`, `mason`, `beholder`, `villager` 계열은 짧아도 무방하지만, 역할 정체성은 남겨야 한다.
- 테마를 넣는다고 해서 조작 동사를 빼지 않는다. `확인해라`, `바꿔라`, `돌려라`, `기억해라`, `눈을 감아라`는 끝까지 남긴다.

## 공통 역할 해석

아래 표는 모든 테마 시나리오에서 유지할 공통 역할 기능이다. 시나리오별 표는 이 공통 기능 위에 감각 어휘와 톤을 덧입힌 것이다.

| 역할 | 공통 말투/태도 | 밤 뒤에 남겨야 할 인상 |
| --- | --- | --- |
| 도플갱어 | 능청스럽고 모방을 즐김 | 남의 주장, 말투, 정체를 빌릴 수 있는 존재 |
| 늑대인간 | 압박형, 사냥감 추적형 | 먼저 몰아가거나 냄새 맡듯 반응을 읽는 존재 |
| 태초의 늑대인간 | 지배형, 판을 짜는 우두머리 | 사람 하나를 바꾸어 판을 설계하는 존재 |
| 신비한 늑대인간 | 속삭이는 심문형 | 상대 반응을 읽고 약점을 캐는 존재 |
| 잠자는 늑대인간 | 무지 연기형 | 정보가 없지만 무지한 척 버티는 존재 |
| 하수인 | 음습한 충성형 | 직접 나서지 않고 시선을 돌리는 존재 |
| 종자 | 수동적 관찰형 | 순서와 위치를 오래 기억하는 보조자 |
| 프리메이슨 | 건조한 사실형 | 감정보다 확인 가능한 사실을 중시하는 존재 |
| 어떤것 | 촉각 중심, 감각형 | 눈보다 접촉과 감촉을 믿는 존재 |
| 예언자 | 차분한 진실 제시형 | 작은 정보 하나로 판을 가르는 존재 |
| 견습 예언자 | 조심스러운 조각 수집형 | 불완전한 단서를 끝까지 품는 존재 |
| 심령 수사관 | 염세적 현장 수사형 | 초자연보다 인간 행동을 의심하는 존재 |
| 강도 | 능글맞은 폭로형 | 남의 정체와 서사를 벗겨 내는 존재 |
| 마녀 | 냉소적 조작형 | 두려움, 소문, 유혹을 이용하는 존재 |
| 말썽쟁이 | 즐기는 교란형 | 작은 어긋남으로 판을 뒤집는 존재 |
| 동네 얼간이 | 엉뚱하지만 핵심을 보는 존재 | 남들이 안 보는 단순한 흔적을 집는 존재 |
| 주정뱅이 | 흐릿하지만 단서를 쥔 존재 | 믿기 어렵지만 이상하게 기억이 남는 존재 |
| 영기 예언자 | 초연한 흐름 감지형 | 누가 움직였는지 기운으로 읽는 존재 |
| 주시자 | 표정 읽는 심문형 | 말보다 얼굴과 망설임을 보는 존재 |
| 계시자 | 최종 폭로형 | 결국 누군가를 드러내는 존재 |
| 불면증 환자 | 끝까지 보는 목격자형 | 마지막 장면을 알고 있다는 존재 |
| 사냥꾼 | 단호한 압박형 | 낮에 투표 방향 자체를 위협하는 존재 |
| 무두장이 | 자해적 도발형 | 일부러 수상해져야 하는 존재 |
| 마을 주민 | 담백한 정리형 | 정보는 없지만 모순을 세는 존재 |
| 왕자 | 신분 의식형 | 쉽게 밀리지 않는 존재 |
| 저주받은 자 | 불안한 오판 유발형 | 보이는 것과 실제가 어긋나는 존재 |
| 견습 무두장이 | 애매한 추종형 | 독립 승리와 마을 플레이 사이에 걸친 존재 |

## 시나리오별 역할 말투/태도

아래 표는 현재 대사 기준으로 읽히는 `역할별 톤`을 정리한 것이다. 실제 수정 시에는 사건 설명을 늘리기보다 이 톤을 더 짧고 선명하게 유지하는 쪽으로 간다.

### 1. beginner_dark_fantasy

방향 요약:

- 유일한 `규칙 학습형` 시나리오다.
- 연기보다 `명확한 지시`, `실수 방지`, `승리 조건 상기`가 우선이다.
- 말투는 캐릭터형이 아니라 `교과서형 진행자`에 가깝게 유지한다.

| 역할 | 말투/태도 | 수정 메모 |
| --- | --- | --- |
| 도플갱어 | 정중한 설명형, 복제 절차를 끝까지 안내 | 복제 후 행동 타이밍을 가장 분명하게 유지 |
| 늑대인간 | 중립 규칙 안내형 | 동족 확인과 고독 늑대 예외를 짧고 정확하게 유지 |
| 태초의 늑대인간 | 강제 교환 안내형 | `반드시` 교환해야 한다는 점을 선명하게 유지 |
| 신비한 늑대인간 | 선택형 확인 안내 | 플레이어 확인만 가능하다는 제약을 분명히 유지 |
| 잠자는 늑대인간 | 소극적 대기 안내 | 눈을 뜨지 않는다는 점과 엄지 신호만 간단히 유지 |
| 하수인 | 협조 절차 안내형 | 늑대 확인 순서만 명확하게 유지 |
| 종자 | 하수인과 유사한 보조 안내형 | 늑대 확인 구조를 하수인과 혼동되지 않게 유지 |

## 바로 적용할 수정 체크리스트

- 각 시나리오의 `openingClips`를 2문장 이내 상황 프레임으로 줄인다.
- 각 역할 `roleClips`는 `개성 대사 > 개성 있는 눈을 떠라 > 행동 > 플레이 규칙 상세 > 개성 있는 눈을 감아라 > 개성 있는 마무리` 흐름으로 다시 정리한다.
- 각 역할 대사에서 사건 설명 문장을 1개 이하로 제한한다.
- 각 역할 대사에 `낮 토론용 훅`을 최소 1개 넣는다.
- `ep1 > ep2 > ep3` 순서로 문장 수와 설명량이 분명히 줄어들게 다시 쓴다.
- 무행동 역할도 `어떤 모순을 보라`는 식의 사회적 임무를 남긴다.
